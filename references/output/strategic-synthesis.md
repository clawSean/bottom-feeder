# strategic-synthesis

After completing individual topic files, generate synthesis documents that connect them into actionable plans.

## Why This Matters

Individual knowledge files are ingredients. Synthesis documents are the meal. A team leader doesn't want to read 30 separate files — they want: "Given everything we now know, what should we do, in what order, and who owns it?"

## Synthesis Document Types

### 1. Priority Matrix
Connect all topic files into a single prioritized action plan.

Structure:
- **Critical Path** (do this week — blocks everything else)
- **High Priority** (do this sprint — enables team velocity)
- **Important** (do this month — builds competitive advantage)
- **Strategic** (do this quarter — positions for next year)

For each item: action, owner, dependency, knowledge file reference.

### 2. Release Scope Recommendation
When topic files cover multiple initiatives for a specific release:
- **Must Ship** — non-negotiable for this release (with rationale)
- **Best Effort** — ship if ready, roll if not (with effort estimates)
- **Out of Scope** — explicitly deferred (with reasoning)
- **Capacity Check** — do we have enough hours?

### 3. Department Playbooks
(See `references/topic-selection/department-playbooks.md` for full spec)

### 4. Competitive Positioning
Synthesize competitive landscape + regulatory + product strategy into a narrative:
- What is our moat?
- Where are we winning?
- Where are we falling behind?
- What should we do about it?

### 5. Monday Action Items
Distill everything into a 1-page quick reference for the next team meeting:
- Top 5 decisions needed
- Key numbers to have ready
- Pre-meeting reading list (links to 3-4 most important knowledge files)
- Post-meeting action items template

## When to Generate

Synthesis documents should be written AFTER the main topic burn, not during. They require the full context of all completed topics to be most useful.

**Recommended timing:**
- Complete 80%+ of topic files first
- Then spend the final 20% of burn time on synthesis
- Quality of synthesis > quantity of additional topics

## Quality Gate

A synthesis document passes when:
- [ ] Every recommendation links to a supporting knowledge file
- [ ] Every action item has an owner name
- [ ] Every priority has a timeline (this week / sprint / month / quarter)
- [ ] The document is useful WITHOUT reading any other files (self-contained summary)
